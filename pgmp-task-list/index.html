# 喜団 (きだん) ベストチョイス

# 【開発業種】

販売

# 【プロジェクト概要】

お客様に手頃な価格の商品を提供することを目的としたシステムの開発。

# 【技術構成】

  • OS: Linux
  • 言語: Java、GRPC など
  • データベース: MySQL Cluster、Redis Cluster
  • ツール: Spring Cloud、Reactor、Spring MVC、Mybatis Plus など
  • その他:
    • メッセージキュー (MQ): RocketMQ Cluster
    • トレースログ: SkyWalking
    • トランザクション管理 (TX): Seata
    • 設定管理: Nacos
    • ログ管理: HDFS、Elastic Search Cluster、FileBeat、Kibana など

# 【業務内容】

  • プロジェクト管理計画書の作成
  • チームビルディング
  • 要件定義・システムアーキテクト構築
  • 日常的なプロジェクト管理
  • 注文・商品モジュールの開発および再構築
  • クーポン・ライブ配信モジュールの構築
  • ゲートウェイの設計・実装
  • CRM システムの認証モジュール構築
  • 技術的課題の解決
  • プロジェクトのクローズ
  • 教訓共有
  • 保守および機能更新

# 【実績・取り組み】

  • 商品モジュールの再構築

    • インターフェースの効率を 20 倍以上向上させ、コードの簡潔性・可読性・拡張性・保守性を改善。
    • Strategy パターンを適用し、異なる市場への対応を強化。
    • SKU の組み合わせ処理を 5 重ループから 2 重ループに簡略化。
    • SPU データを分離し、インメモリデータベースへ格納。

  • データ層の最適化

    • Chain of Responsibility パターンを導入し、DB 層を分離。今後のデータセンター拡張や DB の増減を容易に。

  • 注文モジュールの再構築

    • Strategy、Chain of Responsibility、Filter などのデザインパターンを活用し、コードの簡潔性・可読性・拡張性・保守性を向上。

  • テスト効率の向上

    • 自動化テストの導入を支援し、インターフェーステストの効率向上とコスト削減を実現。

  • トレース（APM）システムの導入

    • 開発部門における問題発見を容易にし、解決時間を大幅に短縮。

  • 開発手法の改善

    • 会社の成長に伴い、ニーズ管理サイクルを標準化。
    • 開発手法をアジャイル開発からハイブリッド開発・ウォーターフォール開発へ移行するサポートを実施。

  • 教訓共有と育成

    • 教訓と技術を共有し、開発部門全体の技術レベル向上を支援。

自己 PR:

中国で 7 年間、様々な規模のプロジェクトに携わってきました。上流から下流までの工程を経験し、特定の業務領域にとらわれず、多岐にわたるシステム案件に貢献してきました。これらの経験を通して、PMP の資格を取得し、プロジェクト管理の知識を深めました。また、顧客折衝から要件定義、設計、開発、テスト、リリース、保守まで、プロジェクト全体を包括的に管理した経験もあります。

日本では SE として、主に設計を担当し、一部テストも経験しました。得意な開発言語は Java ですが、Go や Scala など、新しい言語にも積極的に挑戦しています。

これまでの経験と知識を活かし、PM としてプロジェクトを成功に導く自信があります。
